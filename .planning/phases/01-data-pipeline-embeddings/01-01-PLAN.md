  <action>
Create `backend/ml/features.py` implementing the feature extraction pipeline. Per user decisions:

1. **extract_all_player_map_features(db_path)**: SQL query joining player_map_stats + matches + players (use LEFT JOIN for players table since not all players have entries). Filter WHERE map_name IS NOT NULL AND kills IS NOT NULL. Return list of dicts. Use LOWER() on player name comparisons (Pitfall 3).

2. **AGENT_ROLES dict**: All 29 agents mapped to 4 roles (duelist, initiator, controller, sentinel). Use the exact mapping from RESEARCH.md.

3. **clean_team_name(raw)**: Strip event suffixes using regex pattern matching year (20XX) or VCT/Champions Tour keywords. Mirror the pattern in Database._clean_team_name().

4. **compute_team_win_records(rows)**: Iterate all rows, parse map_score ("A-B" format), determine if player's team won. Aggregate wins/total per team (cleaned names). Return dict of {team_lower: {wins, total}}.

5. **compute_rolling_features(player_name, current_event_id, player_history, window=10)**: Filter player_history to rows with event_id < current_event_id (STRICTLY causal). Sort by event_id descending, take top `window`. Compute rolling averages for: kills, deaths, assists, acs, adr, kast, first_bloods. If no prior records exist, return None (caller handles with global means).

6. **build_feature_matrix(db_path)**: Orchestrate the full pipeline:
   - Extract all raw rows
   - Compute global mean stats (from all rows, for imputation)
   - Group rows by player_name (lowercased) for efficient rolling lookups
   - Compute team win records
   - For each row: compute rolling features (impute with global means if None), compute opponent_win_rate, add agent_role, assemble complete feature dict
   - Return list of feature dicts ready for Dataset consumption

Each feature dict must contain: player_name, map_name, agent, agent_role, kills (target), deaths, assists, acs, adr, kast, first_bloods, event_id, match_id, opponent_win_rate, plus rolling_* features (rolling_kills, rolling_deaths, rolling_assists, rolling_acs, rolling_adr, rolling_kast, rolling_first_bloods).

Use `logging.getLogger(__name__)` for logging. Use type hints from `typing` module per project conventions.
